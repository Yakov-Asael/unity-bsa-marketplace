## Template 1 — Project / meeting review deck ("Enrichment")
Use for project reviews, retrospectives, feature wrap-ups, stakeholder readouts. Slide order:
1. **Title** — project name + date
2. **Agenda**
3. **Project Goal**
4. **Business requirement / What needs to be solved**
5. **Solution options & constraints**
6. **Who it serves** — which divisions
7. **Issues during the project**
8. **Demo** (live example placeholder)
9. **What's the effort of support today**
10. **What could be done differently**
11. **How it was measured / monitored**
12. **Thank you**

## Template 2 — General "AI-BSA" style deck
Use for training, concept, or topic presentations. Structure:
- **Title + subtitle**
- **Agenda** — numbered sections (01, 02, 03…)
- **Section divider** slides between major sections
- **Content** slides: short title + a few bullets (one idea each)
- **Examples** slides where useful
- **Next Steps**
- **Closing**

## Unity deck rules (always)
- **Palette:** brand blue `#2196F3`, near-black `#141514`, white, grays. No other accent colors. (The exported template already carries Unity branding — keep it.)
- **Not swamped:** one idea per slide; max ~5 bullets; short phrases, not paragraphs; generous white space.
- **Visual hierarchy:** clear slide title, then supporting points. Lead with the point.
- Speaker-facing detail goes in **speaker notes**, not on the slide.
- End decks with a clear closing / Next Steps.
