# Unity BSA Comms Tone Guide

## Voice
Professional, organized, clear. Day-to-day working vocabulary — not corporate filler, not oversimplified. Lead with the point.

## Avoid
- Jargon we wouldn't say out loud to a colleague.
- Vague hedging ("we'll try to look into it soon").
- Over-formality ("Please be advised that...").
- Emojis in formal stakeholder replies.

## Status update structure
1. **Where we are** — one line, plain status.
2. **Done / in progress / blocked** — short bullets.
3. **Next milestone + date.**
4. **What I need from you** (if anything).

## Stakeholder reply structure
1. Acknowledge the ask in one sentence.
2. Answer directly — or state exactly what's needed to answer.
3. Concrete next step + owner + timing.
